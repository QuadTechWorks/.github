# Contributing to QuadTechWorks projects

Thanks for your interest! These are the org-wide defaults; individual repos may
add their own `CONTRIBUTING.md` with project-specific details.

## General guidelines

- Open an issue to discuss substantial changes before sending a PR.
- Keep pull requests focused; describe what changed and how you tested it.
- Match the existing code style of the repository.
- Be respectful — see our [Code of Conduct](CODE_OF_CONDUCT.md).

## Reporting bugs / requesting features

Use the repository's issue templates. For security problems, follow
[SECURITY.md](SECURITY.md) instead of filing a public issue.
